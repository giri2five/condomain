# condom@in Privacy Policy

Last updated: March 29, 2026

## Overview

`condom@in` is a browser extension that helps protect users from wallet phishing and malicious signing flows by creating per-site proxy wallet identities inside the browser.

## What data the extension stores

The extension stores the minimum data it needs to operate, including:

- site-specific proxy wallet metadata
- encrypted proxy private key material required for those proxy wallets
- extension state such as whether protection is enabled
- local activity and threat history used to show protection events in the popup
- rotation and usage metadata for site-specific proxies

## How data is stored

- data is stored locally in the browser using extension storage
- proxy private key material is stored in encrypted form
- the extension is designed to operate locally inside the browser

## What data is not collected by us

`condom@in` does not intentionally collect, sell, or transmit personal data to a remote company server as part of its core protection flow.

The extension does not require an account.

## Website access

The extension requests access to websites so it can intercept wallet provider flows and protect users on supported dapps and pages.

## Notifications

The extension may show local browser notifications when it detects risky signing behavior or threat-like patterns.

## Security note

No browser extension can guarantee total protection against all wallet loss scenarios. Users should still avoid sharing seed phrases, installing untrusted software, or approving unknown wallet actions.
